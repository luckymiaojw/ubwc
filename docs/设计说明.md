# UBWC 设计说明

## dec metadata command generator

### 模块范围

`src/dec/ubwc_dec_meta_get_cmd_gen.v` 中的 `ubwc_enc_meta_get_cmd_gen` 负责为 decoder metadata 读取产生命令和对应的 metadata 描述信息。

该模块每次产生一个 64-bit metadata 读取命令，也就是一次固定读取 8 个 tile 的 metadata byte。`cmd_len` 不再由该模块输出，上层固定按 8 bytes 处理。

### 接口行为

模块输出一组 metadata group 接口：

- `meta_grp_valid/meta_grp_ready/meta_grp_addr`：输出 metadata group 读地址
- `meta_format/meta_xcoord/meta_ycoord`：输出与本次读地址对应的 metadata 描述

`meta_grp_valid` 有效表示当前坐标、格式和地址已经计算完成，并且 `meta_grp_addr/meta_format/meta_xcoord/meta_ycoord` 属于同一组数据。

模块内部使用：

```verilog
issue_fire = meta_grp_valid && meta_grp_ready;
```

只有当 `meta_grp_valid` 和 `meta_grp_ready` 同时有效时，`issue_fire` 才成立。模块在 `issue_fire` 的同一个时钟沿推进到下一个坐标。若 ready 无效，当前坐标、地址和格式保持不推进。

### 扫描计数方式

该模块不使用显式状态机，而是使用以下寄存器驱动扫描：

- `scan_active`：当前 frame metadata 扫描进行中
- `xcoord_cnt`：当前 x 方向 tile 坐标，每次推进 8 个 tile
- `y_row_cnt`：RGBA 或 Y plane 的 metadata 行计数
- `uv_row_cnt`：UV plane 的 metadata 行计数
- `scan_is_uv_plane`：当前输出是否属于 UV plane

`start` 有效时，模块清零上述计数器，并从第一组坐标开始计算。如果格式不支持或 tile 宽高为 0，则不启动扫描。

### 格式扫描顺序

当前支持：

- `RGBA8888`
- `RGBA1010102`
- `YUV420 8-bit`
- `YUV420 10-bit`

当前不支持 `YUV422`，该格式不会启动有效扫描。

RGBA 格式按普通逐行扫描：

```text
RGBA row0
RGBA row1
RGBA row2
...
```

YUV420 格式按如下顺序扫描：

```text
Y row0
Y row1
UV row0
Y row2
Y row3
UV row1
...
```

也就是每完成两行 Y metadata，再读取一行 UV metadata。对于奇数高度，最后一行 Y 完成后仍会读取对应的最后一行 UV。

### 地址计算

metadata 地址计算与 encoder 侧 `ubwc_enc_meta_addr_gen.sv` 保持一致。

当前坐标先决定使用哪个 base：

- RGBA 和 Y plane 使用 `meta_base_addr_rgba_y`
- UV plane 使用 `meta_base_addr_uv`

行 base：

```verilog
meta_y_base_addr = current_ycoord[TH_DW-1:4] * {meta_pitch_bytes, 4'd0};
```

其中：

```verilog
meta_pitch_bytes = (tile_x_numbers + 63) & 16'hffc0;
```

tile 内 offset：

```verilog
meta_xy_offset_addr = {
    xcoord_cnt[TW_DW-1:4],
    current_ycoord[3],
    xcoord_cnt[3],
    current_ycoord[2:0],
    xcoord_cnt[2:0]
};
```

最终地址：

```verilog
meta_grp_addr = meta_base_addr + meta_y_base_addr + meta_xy_offset_addr;
```

### 输出坐标含义

`meta_xcoord` 和 `meta_ycoord` 输出的是当前 8-tile metadata 读取块对应的 tile 坐标：

- `meta_xcoord`：当前读取块的起始 x tile 坐标，取值为 `0, 8, 16, ...`
- `meta_ycoord`：当前读取块所在的 Y/RGBA 或 UV metadata 行坐标

`meta_format` 输出 tile-level format：

- RGBA：`META_FMT_RGBA8888` 或 `META_FMT_RGBA1010102`
- YUV420 8-bit：Y 输出 `META_FMT_NV12_Y`，UV 输出 `META_FMT_NV12_UV`
- YUV420 10-bit：Y 输出 `META_FMT_P010_Y`，UV 输出 `META_FMT_P010_UV`

### 完成条件

当最后一个有效坐标在 `issue_fire` 下完成输出后，`scan_active` 拉低，内部 `frame_done` 拉高。`frame_done` 当前主要用于 testbench 判断单帧扫描完成，设计对外仍通过 metadata group handshake 表达有效输出。

## dec metadata AXI read command generator

`src/dec/ubwc_dec_meta_axi_rcmd_gen.v` 中的 `ubwc_dec_meta_axi_rcmd_gen` 接收 metadata group 请求，并负责完成 AXI 读、256-bit beat 内 64-bit lane 选择，以及 byte 级 metadata 输出。

### 读命令对齐

`meta_grp_addr` 是 64-bit metadata group 的真实 byte 地址。模块对 AXI AR 地址做 256-bit 对齐：

```verilog
aligned_cmd_addr = {meta_grp_addr[ADDR_WIDTH-1:5], 5'd0};
```

因为 AXI RDATA 宽度为 256 bit，也就是 32 bytes；每次只需要其中一个 64-bit metadata group，所以 lane 选择来自：

```verilog
cmd_lane_sel = meta_grp_addr[4:3];
```

AXI 读固定为单 beat：

```verilog
m_axi_arlen = 8'd0;
```

### 64-bit group 到 8-bit stream

AXI RDATA 返回后，模块先根据 `cmd_lane_sel` 从 256-bit 数据中取出有效的 64-bit metadata group。

对外输出时不再直接输出 64-bit group，而是拆成 8 个 8-bit metadata byte：

```text
byte0 -> meta_data_xcoord[2:0] = 3'd0
byte1 -> meta_data_xcoord[2:0] = 3'd1
...
byte7 -> meta_data_xcoord[2:0] = 3'd7
```

`meta_data_format` 和 `meta_data_ycoord` 在 8 个 byte 输出期间保持不变；`meta_data_xcoord` 等于 group 起始 x 坐标加当前 byte index。

下游 `ubwc_dec_meta_axi_rdata_to_sram.v` 接收该 8-bit stream 后，会重新收集 8 个 byte，组成一个 64-bit word，再按 row phase 写入 metadata ping-pong SRAM。
